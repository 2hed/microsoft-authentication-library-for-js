Authentication Options

Type declaration

  • Optional authority?: string

    Follow up with the authority discussion with the PMs Until then this is a placeholder

  • clientId: string

    Client ID assigned to your app by Azure Active Directory.

  • Optional navigateToLoginRequestUrl?: boolean

    Used to turn off default navigation to start page after login. Default is true. This is used only for redirect flows.

  • Optional postLogoutRedirectUri?: string | function

    Used to redirect the user to this location after logout. Defaults to window.location.href.

  • Optional redirectUri?: string | function

    The redirect URI of the application, this should be same as the value in the application registration portal. Defaults to window.location.href.

  • Optional state?: string

    Use to send the state parameter with authentication request

  • Optional validateAuthority?: boolean

    Used to turn authority validation on/off. When set to true (default), MSAL will compare the application"s authority against well-known URLs templates representing well-formed authorities. It is useful when the authority is obtained at run time to prevent MSAL from displaying authentication prompts from malicious pages.

TFrameworkOptions

TFrameworkOptions: object
  • Defined in Configuration.ts:117

App/Framework specific environment Support

Type declaration

  • Optional isAngular?: boolean

    flag set to determine if it is Angular Framework

  • Optional protectedResourceMap?: Map<string, Array<string>>

    This is mapping of resources to scopes used by MSAL for automatically attaching access tokens in web API calls. A single access token is obtained for the resource. So you can map a specific resource path as follows: {"https://graph.microsoft.com/v1.0/me", ["user.read"]}, or the app URL of the resource as: {"https://graph.microsoft.com/", ["user.read", "mail.send"]}. This is required for CORS calls.

  • Optional unprotectedResources?: Array<string>

    Array of URI's which are unprotected resources. MSAL will not attach a token to outgoing requests that have these URI. Defaults to 'null'.
